Comandos usados no git:

1º:  Criação de uma pasta com o comando mkdir.
2º  Inicialização do repositório com o comando git init.
3º  Uso do comando git config global para sincronizando das contas com o devido usuário corretamente.
4º  Inserindo um arquivo txt ( ao qual está sendo usado para escrever esse relatório nesse exato momento) com o comando touch relatorio.txt.
5º  Git Status para verificar o estado do repositório no atual momento.
6º Git Add para adicionar o arquivo no repositório.
7º Git Commit-m para descrever a mensagem de adicionar arquivo.
8º Git Add. para a pasta chamada projeto_site2 dentro do repositório (ela foi movida para o repositório local Projeto).
9º Git Status para verificar se os arquivos foram adicionados corretamente.
10º Git add relatório.txt no repositório local.
11º Git Remote Add Origin Master para mapeamento (apontar) no repositório remoto no GitHub.
12º Git Push para o envio dos arquivos no repositório criado remotamente.
13º Git Status para verificar a situação do repositório local.
14º Git Add. para adicionar todos os arquivos situados numa mesma pasta, devido a modificações de css na página.
15º Git commit -m demarcando a mensagem de alteração dos arquivos adicionados.
16º Git push origin master para envio dos arquivos no repositório remoto do Git Hub.
17º Git clone para baixar os arquivos no repositório local.
18º Git Add Projeto_Site2 devido a alterações pedida pelo cliente no gradiente background usando CSS.
19º Git Commit -m para descrever as alterações solicitadas pelo cliente.
20º Git Push para envio dos arquivos no repositório remoto.
21º Git Add Projeto_Site2 devido a alterações no código CSS.
22º Git Git Commit-m desfazendo as alterações, pois o cliente rejeitou as alterações realizadas no gradiente da página.
23º Git Push para envio dos arquivos para o repositório remoto.
24º Git Status no repositório local para verificação do estado.
25º Git Add relatorio.txt.
26º Git commit -m "Acrescentando comandos"
27º Git Push para envio dos arquivos.

OBS: Houve durante o processo de realização da atividade vários comandos repetidos como Git Status,Git Push que foram omitidos, em parte, para não ficar muito repetitivo os comandos descritos neste relatório.
